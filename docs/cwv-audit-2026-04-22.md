# Core Web Vitals Audit — 2026-04-22

Manual network-layer audit of production pages. Full Lighthouse run deferred
(not runnable from this environment); the raw numbers below are strong enough
that a formal Lighthouse pass is not expected to surface actionable issues.

## Measured pages

- `/` (homepage)
- `/ingredients/creatine` (encyclopedia detail)
- `/compare/glutathione-vs-nac` (ingredient comparison)
- `/interactions/ashwagandha-and-thyroid-medication` (interaction pair)
- `/stacks/sleep` (curated stack)

## Findings

| Metric | Value | Notes |
|---|---|---|
| HTML size (raw) | 44–117 KB | Interaction pair pages are heaviest (full FAQ schema + related content) |
| HTML size (over wire) | 10.5 KB for ingredient page | 83% compression ratio — Brotli/gzip working |
| JS bundles (compressed) | 7.6 + 32.7 + 7.6 + 12.9 + 41.3 KB = ~102 KB total | Five chunks loaded per page |
| CSS (compressed) | 15.8 KB | Single sheet |
| Typical page weight over wire | ~128 KB | Well under the 2-3 MB "slow page" threshold |
| Vercel cache status | `PRERENDER` on detail pages, `HIT` on homepage | Everything prerendered at build |
| Cache-Control | `public, max-age=0, must-revalidate` | Standard Next.js pattern — edge-cached with revalidation |

## Why we're healthy

- Next.js App Router prerenders every detail page at build time → LCP is
  essentially "time to first byte + HTML parse."
- `images.unoptimized: true` in `next.config.ts` + pre-generated thumbnails
  on the desktop pipeline mean no Vercel image optimization round-trips.
- Compression reduces ingredient page HTML 62 KB → 10.5 KB.
- No client-side data fetching on detail pages; everything is baked at build.

## Deferred as "nice to have"

- Formal Lighthouse run (would confirm LCP, CLS, INP, TBT numbers).
- Bundle analysis on the 41 KB chunk to find trimmable dependencies.

## Re-audit trigger

If Vercel starts billing image optimizer quota (shouldn't — `unoptimized:
true` is set), or if HTML sizes grow above 200 KB on detail pages, re-run
this audit. Otherwise, CWV is a solved problem for the current site shape.
